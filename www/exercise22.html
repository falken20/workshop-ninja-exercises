Vamos ahora a revisar el metodo que se ejecuta cuando damos de alta un Ninja, en este caso se ejecuta el método create.

Si ves la primera línea del método lo que se hace es recoger todos los campos del formulario y guardarlos en la variable ninja_data en formato diccionario de Python (clave:valor): 

{'c1': 'v1', 'c2': 'v2', ...}

¿Serías capaz de mostrar su contenido en las trazas usando logging?. Aparte del mensaje en este caso tenemos que mostrar el contenido de una variable, se puede hacer de varias formas pero como pista indicarte que puedes usar signos para operar y como el contenido de la variable ninja_data es un diccionario lo tenemos que formatear con format(ninja_data) para que asi lo pase a string.

Si pruebas y observas bien el log verás tu traza pero aparte también verás un error, intenta ver el motivo del error para ver qué pasa aunque tiene que ver con el siguiente ejercicio.

Dos posibles formas de hacerlo: 
logging.info('Mensaje: ' + format(ninja_data))

logging.info('Mensaje: %s', ninja_data) -> Con %s indicamos que se muestra un string y lo coja del siguiente parametro (ninja_data),
                                           no hace falta formatearlo como en ejemplo anterior ya que lo hace automáticamente
