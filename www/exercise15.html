El fichero main.py es el fichero Python principal de la aplicación, en él y mediante la clase WSGIApplication definiremos la aplicación y mediante la clase Route estableceremos la definición de cada una de las rutas de nuestra app, indicando para cada ruta su path, el handler (clase), el metodo dentro de ese handler que se ejecutará así como el método HTTP permitido (GET, POST, PUT, etc.).

Revisa los comandos import al inicio del fichero así como las rutas declaradas y localiza los ficheros Python involucrados así como las clases y metodos dentro de estos. 

./src/ninjas.py
    Clase: Ninjas
    Métodos: list, create, retrieve, update, delete
./src/moocs.py
    Clase: Moocs
    Métodos: list, create, retrieve, delete
./src/ranking.py
    Clase: Ranking
    Métodos: ranking, points

Sobre las rutas y métodos HTTP empleados ten en cuenta que se ha definido un API REST, la cual establece una convención acerca de la nomenclatura de las rutas y los métodos HTTP empleados, de manera que cada ruta y método definen una acción concreta sobre un recurso.

En nuestro caso, sobre el recurso Ninja, nuestra API tiene las siguientes acciones:

    GET /ninjas: Devuelve el listado de ninjas existente
    GET /ninjas/<id>: Devuelve la información del ninja cuyo identificador se indica, o un código HTTP 404 si no existe
    POST /ninjas: Crea un nuevo ninja a partir de la información enviada en el cuerpo de la petición HTTP
    PUT /ninjas/<id>: Modifica la información del ninja cuyo identificador se indica con la enviada en el cuerpo de la petición HTTP, o devuelve un código 404 si no existe
    DELETE /ninjas/<id>: Elimina el ninja cuyo identificador se indica, o devuelve un código HTTP 404 si no existe
