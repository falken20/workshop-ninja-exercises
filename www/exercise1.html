Clonado fuentes

Vamos a bajarnos el código del repositorio para poder realizar las prácticas. Como repositorio hemos usado GitHub y lo que vamos a hacer es clonar el proyecto en nuestra máquina. Sitúate en la carpeta que decidas y ejecuta el siguiente comando: 

git clone https://github.com/richionline/Workshop-Ninja-Python.git

Esto nos creará una carpeta con el nombre del proyecto y todos los ficheros dentro. Valida que se ejecuta correctamente el clonado y accede a la carpeta creada. 

  $ git clone https://github.com/richionline/Workshop-Ninja-Python.git
  ...
  Receiving objects: 100% (4743/4743), 17.54 MiB | 4.90 MiB/s, done.
  Resolving deltas: 100% (897/897), done.
  $ cd Workshop-Ninja-Python
Sobre la estructura de proyecto destacamos los siguientes ficheros y carpetas: 
  ./www Carpeta que contiene todos los ficheros que componen el front de la aplicación
  ./src Carpeta que va a contener nuestros fuentes Python
  app.yaml Archivo que nos va a permitir configurar nuestra aplicación. Por ejemplo indicar entorno ejecución, rutas URL, etc.
  index.yaml Archivo donde vamos a definir los índices para las consultas complejas
  main.py Archivo Python que va a ser el principal de la aplicación. En él definiremos la aplicación y rutas
  appengine_config.py Archivo Python para poder personalizar los módulos de Python, con ello podemos anular (sobreescribir)
                             constantes y funciones existentes.
  requirements.txt Contiene las dependencias de paquetes que vamos a necesitar
  .gcloudignore Especifica los archivos que no se van a desplegar en Google Cloud Platform
  .gitignore Especifica los archivos que no se gestionan en repositorio Git
