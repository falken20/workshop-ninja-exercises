Anteriormente hemos configurado el nombre del servicio (service) en el fichero app.yaml, lo que haremos a continuación es usar ese parámetro como nombre del namespace que cada uno va a usar.

Con este namespace lo que conseguimos es que sobre un mismo sistema de almacenamiento, en este caso datastore, cada usuario tenga su propio espacio definido donde se guardaran sus entidades sin interferir con las del resto.

Abre el fichero config.py y detecta la línea donde se recoje el nombre del servicio definido para asignarlo a la variable donde guardaremos el namespace a utilizar.

NOTA: Aunque ahora se llamen servicios Google los denominaba inicialmente módulos, pero esta nomenclatura no ha cambiado en el código, que mantiene el nombre antiguo. 

NAME_SPACE = modules.get_current_module_name()
El método get_current_module_name lo que nos devuelve es el nombre del servicio definido en app.yaml. La API de Modules proporciona funciones que muestran información acerca del entorno operativo (módulos o servicios, versiones e instancias)
